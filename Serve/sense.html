Serve Sense

صورة العطل أو صوت العميل ← تشخيص ← 3 حلول مسعّرة ← فني — في 60 ثانية.

معمارية من طبقتين: بوّابة استقبال رخيصة وسريعة تفرز، وعقل دقيق يُستدعى عند الحاجة لتقدير يمنع خسارة الشركة أو العميل — مُرسىً على قاعدة بيانات أسعار محلية تجعل التقدير واقعياً لا عشوائياً، مع وحدة صيانة ذاتية آمنة.

⚙️ معرّفات المودلات والأسعار مُتحقَّق منها من مرجع Anthropic الرسمي (claude-api). «Claude 3.5 Sonnet» قديم — البديل الحالي Sonnet 4.6 / Opus 4.8.

إجابة السؤال — أي خطوة أولاً؟

✅ هجين: الاثنان معاً — الموجّه و قاعدة الأسعار، لأنهما لا يعملان منفصلين.

① الموجّه واللهجة

يعطي النموذج اللسان والحُكم: فهم العامّية، تحديد الخطورة، صياغة الحل — لكنه بلا أسعار حقيقية يبقى يخمّن.

② قاعدة الأسعار (RAG)

تعطيه الحقيقة المحلية: 280–450 ر من بيانات السوق لا من خياله. لكن بلا موجّه ذكي لا يعرف متى وكيف يستخدمها.

③ الربط = القيمة

الموجّه يأمر النموذج صراحةً: «لا تذكر سعراً إلا من الجدول المسترجَع». هنا فقط يصبح التشخيص دقيقاً ولا يُنسخ.

القاعدة العملية: ابدأ بالموجّه + سُلّم الخطورة (يعمل فوراً ويُثبت القيمة)، وبالتوازي ابنِ قاعدة الأسعار واربطها كمصدر الحقيقة الوحيد للتقدير. الإرساء (Grounding) هو الجسر بينهما.

01

المعمارية — بوّابة استقبال + عقل

رخيصة سريعة تفرز، وعقل غالٍ دقيق يُستدعى فقط عند الحاجة.

L0تطبيع المدخلات: الصوت ← نص (تحويل صوت لنص خارج النموذج)، الصورة ← base64.
↓
L1بوّابة الاستقبال (Gatekeeper) — Claude Haiku 4.5رؤية + عربية + سرعة

تصنيف العطل (الخدمات الـ8) + استخراج الكلمات المفتاحية + درجة ثقة + درجة خطورة أوّلية — مخرجات JSON صارمة. لا تفكير عميق (Haiku لا يدعم effort) — مجرّد فرز سريع.

↓ منطق التوجيه (Router) — كود لا نموذج
مسار سريع (روتيني / ثقة عالية)

ردّ مباشر من L1 + 3 حلول من كتالوج التسعير المحلي. ~80% من الحالات. أرخص وأسرع مسار.

تصعيد (معقّد / خطر / تقدير حسّاس)

يُمرَّر النصّ المستخرج إلى العقل.

↓
L2العقل (Brain) — Claude Sonnet 4.6تفكير تكيّفي + effort

تشخيص دقيق + 3 حلول مسعّرة + تقدير يمنع الخسارة + فحص قبول الصيانة الذاتية. مُرسىً على الجدول المسترجَع.

↓ أصعب ~5% (ثقة منخفضة/سلامة)
L3 — العقل الأعمق · Opus 4.8

للحالات الأصعب فقط (effort: high) — أعلى دقّة، أعلى كلفة.

L4 — إنسان في الحلقة (HITL)

مشرف فنّي يراجع قبل الإسناد عند الثقة المنخفضة أو إشارة سلامة. شبكة أمان نهائية.

02

الموجّه الأساسي — لهجة سعودية + سُلّم خطورة

اللسان الذي يفهم «المكيف يقطّر موية» ويصنّف الخطر بدقّة.

سُلّم الخطورة — 5 مستويات

المستوىالرمز البرمجيالوصفالمهلةمثال (تكييف/سباكة)
🟢 آمنsafeلا عطل فعلي / سوء استخدامإرشاد ذاتيالريموت بدون بطارية
🔵 عاديnormalعطل لا يهدّد السلامة، يحتمل التأجيل48–72 سالمكيف ما يبرّد كفاية
🟡 عاجلurgentيتفاقم أو يسبّب ضرراً مادياً4–12 سالمكيف يقطّر موية بغزارة
🟠 طارئemergencyخطر تلف ممتلكات وشيك1–3 سانفجار ماسورة وغمر الغرفة
🔴 خطر — سلامةhazardخطر مباشر على الأرواحفوري — أوقف واتّصل بالطوارئريحة غاز / دخان / أسلاك حيّة + ماء
⚖️ قاعدة الترجيح: عند الشك بين مستويين، اختر الأعلى خطورة (مبدأ الحيطة). أي إشارة لغاز/دخان/نار/أسلاك مكشوفة/ماء يلامس كهرباء ← hazard فوراً.

قاموس القطع المحلية (داخل الموجّه)

مصطلحات لا توجد في أي قاموس رسمي — جوهر الخندق.

  • • كوع = وصلة مواسير بزاوية
  • • يلبو = كوع كبير/وصلة صرف منحنية
  • • فريون أمريكي/هندي = أنواع غاز التبريد
  • • طبلون = لوحة القواطع الكهربائية
  • • كمبروسر = ضاغط المكيف/الثلاجة (الأغلى)
  • • سيفون = خزّان الطرد في الحمّام

مقتطف من موجّه النظام

# الهوية واللغة
أنت «سيرف سِنس»، محرّك التشخيص الذكي.
تفهم اللهجة السعودية العامّية ومصطلحاتها
الدارجة، وتردّ بنبرة ودودة مطمئنة، لكن
المخرَج النهائي JSON صارم فقط.

# التسعير (مُرسىً)
مصدرك الوحيد للأسعار هو "جدول الأسعار
المسترجَع" أعلاه. ممنوع تخمين أي رقم.
التقدير = نطاق [min, max] من الصف الأقرب.
إن لم يوجد صف مطابق ← needs_human=true.

المخرَج المهيكل (Structured Output)

// output_config.format بمخطّط JSON صارم — كل كائن additionalProperties:false
{
  "category": "hvac",            // إحدى الخدمات الـ8 أو out_of_scope
  "severity": "urgent",         // safe|normal|urgent|emergency|hazard
  "confidence": 4,               // 1..5
  "likely_cause": "انسداد خرطوم تصريف المكيف",
  "cost_range": { "min": 280, "max": 450, "basis": "قطعة+أجرة" },
  "diy_eligible": true,           // هل يقبل صيانة ذاتية آمنة؟ (انظر §05)
  "solutions": [ /* 3 حلول، كلٌّ مع source_row_id للتتبّع */ ],
  "needs_human": false
}
03

الإرساء — كيف نمنع الهلوسة في السعر

النموذج لا يعرف أي سعر إلا ما نسلّمه له. ثلاث طبقات متراكبة.

1
حقن السياق كحقيقة وحيدة

الجدول المسترجَع يُحقن في الموجّه مع أمر قاطع: «لا تذكر سعراً غير موجود في الجدول».

2
مخرَج صارم بالتتبّع

أداة strict:true تُجبر النموذج على إعادة source_row_id لكل تقدير — لا يستطيع تركه فارغاً.

3
تحقّق بعد الاستدلال

كودك يتأكّد أن السعر يقع داخل نطاق الصف المسترجَع فعلاً — أي انحراف ← تصعيد لإنسان.

# الطبقة 3 — تمسك الهلوسة التي تفلت من الطبقتين الأوليين
for sol in output["solutions"]:
    row = retrieved_by_id.get(sol["source_row_id"])
    if row is None or not (row.price_min <= sol["price_min"]
                           and sol["price_max"] <= row.price_max):
        return escalate_to_human()   # هلوسة محبوطة
💡 لماذا «نطاق» لا «رقم واحد»؟ التقدير 280–450 ر صادق لأنه يعكس تباين السوق الحقيقي. الرقم القاطع الواحد ادعاء دقّة زائفة = أكبر مصدر للخسارة.
04

خندق البيانات — قاعدة متجهية (RAG)

أسعار السوق السعودي + المسميات المحلية. كل تشخيص ناجح يغذّيها فيصعب اللحاق بها.

بنية البيانات · price_items

CREATE TABLE price_items (
  service        TEXT,   -- 8 خدمات
  symptom        TEXT,   -- "المكيف يقطّر موية"
  part_canonical TEXT,   -- "وصلة تصريف مكيف"
  local_synonyms TEXT[], -- {كوع، يلبو، فريون أمريكي} ← الخندق
  price_min      NUMERIC,-- 280.00 (نطاق لا رقم)
  price_max      NUMERIC,-- 450.00
  price_basis    TEXT,   -- "قطعة+أجرة" | "أجرة فقط"
  region         TEXT,   -- الرياض / حي
  last_verified_at TIMESTAMPTZ, -- حوكمة القِدم
  source         TEXT,   -- "فاتورة فني" (ادعاء موثّق)
  sample_size    INT,    -- كم عملية دعمت النطاق
  embedding      vector(1024)
);

الاسترجاع الهجين (إلزامي)

البحث الدلالي وحده يفشل في القطع النادرة («يلبو»)؛ والمعجمي وحده يفشل في إعادة الصياغة («يسرّب» مقابل «يقطّر»). الدمج يلتقط الاثنين.

score = 0.7·cosine(qVec, itemVec)
      + 0.3·lexical(synonyms)   // top-k = 8

⚠️ Anthropic لا توفّر تضميناً — التضمين من مزوّد خارجي للعربية (مثل Cohere متعدّد اللغات، بُعد 1024)؛ الفهم والتسعير من Claude. ثبّت المزوّد — تغييره يُبطل كل المتجهات.

حوكمة + تخزين مؤقت (Prompt Caching)

  • • كل ادعاء سعري له source وlast_verified_at — الأقدم/الأضعف عيّنةً = إشارة ضعيفة.
  • • ترتيب الكاش: جدول الأسعار + الموجّه + أمثلة few-shot أولاً مع cache_control، ثم مدخل المستخدم المتغيّر.
  • • قراءة الكاش ≈ 0.1×، كتابته 1.25× (TTL 5 د). أدنى بادئة: Haiku/Opus 4096، Sonnet 2048 توكن.
05

وحدة الصيانة الذاتية (DIY) — بأمان مُحكَم

تعليمات إصلاح ذاتي — فقط حين يقبل العطل ذلك بأمان تامّ.

✅ يُسمح بالصيانة الذاتية إذا
  • • الخطورة safe أو normal فقط.
  • • إجراء بسيط بلا أدوات خطرة (تسليك مغسلة، تنظيف فلتر، إعادة ضبط، تبديل بطارية).
  • • لا تلامس كهرباء حيّة ولا غاز ولا ماء ساخن تحت ضغط.
  • • ثقة النموذج عالية ووصف العميل واضح.
🚫 يُمنع الـDIY منعاً قاطعاً (إسناد فوري)
  • • أي خطورة urgent/emergency/hazard.
  • • تسرّب غاز · دخان/نار · أسلاك مكشوفة أو جهد عالٍ.
  • • التعامل مع الفريون/الكمبروسر · لوحة الكهرباء (طبلون).
  • • مخاطر إنشائية أو ماء يلامس كهرباء.

منطق القرار

if severity in {"urgent","emergency","hazard"} or hazard_signal:
    diy_eligible = False            # السلامة تتقدّم دائماً
elif category in SAFE_DIY_CATALOG and confidence >= 4:
    diy_eligible = True; emit(diy_steps)  # خطوات + محفّزات توقّف
else:
    diy_eligible = False; dispatch_technician()
مثال يقبل DIY

«مغسلة مسدودة» → خطوات: ماء حار + بيكنج صودا وخل، انتظر 15 دقيقة، كبّاس يدوي. إن استمر ← اطلب سبّاكاً.

محفّزات التوقّف الفوري

ظهور ريحة/حرارة/شرارة، أو فشل الخطوة، أو أي شكّ ← يتوقّف العميل ويُعرض زرّ «اطلب فني الآن».

إخلاء المسؤولية

«إرشادات عامة على مسؤولية العميل؛ سيرف غير مسؤولة عن أضرار الصيانة الذاتية» — يُعرض قبل أي خطوات.

لماذا الـDIY المجاني مصلحة لا خسارة؟

الأعطال التافهة (بطارية ريموت، فلتر) غير مربحة كزيارة أصلاً. حلّها مجاناً عبر Sense يبني ثقة هائلة ويكسب العميل لأول طلب حقيقي مدفوع — مع بوّابة سلامة صارمة تضمن ألّا يلتهم الـDIY أي طلب مربح أو يعرّض أحداً للخطر. تحويلٌ للتكلفة من خسارة إلى أداة اكتساب وبيانات.
